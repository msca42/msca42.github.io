数据类型

  • Java的数据类型分为两种，一个是基本数据类型(值类型，内置数据类型)，一个是引用数据类型。

基本数据类型

  • Java在创建之初就引入了基本数据类型，目的是用来支持数值计算，因为基本数据类型可以在执行效率和内存使用两方面提升软件性能。
  • 基本数据类型有四类八种，分别是整型，浮点型，布尔型和字符型。整型又分为byte,short，int,long四种，浮点型又分为float,double两种，布尔型为boolean,字符型为char。
  • 当我们谈论基本数据类型的时候，我们会从这几个方面去说:应用场景，取值范围(值域)，创建时间，创建空间，类型转换。下面我们一个一个来说。

  • 首先对于数据类型，它的应用场景在于修饰变量。那么创建时间与创建空间也就是变量创建的时间与空间。变量在Java中可以分为这样几类:类变量，实例变量，局部变量。

    • 类变量:独立于方法之外的变量，用static修饰，存放在方法区中，属于共享变量

    • 实例变量:独立于方法之外的变量，没有static修饰，存放在堆中，通过GC回收，属于对象的成员变量，根据访问控制符的不同进行设置

    • 局部变量:类的方法中的变量，声明在方法，构造方法或者语句块中，没有默认值，需要手动初始化,在栈中生成消失

  • 依据赋值给不同类型的变量，基本数据类型的应用场景也限于类，实例和局部这三部分。众所周知，物理层传输的是bit(比特)流，8bit相当于1byte(字节),硬盘中的存储单位为byte。在Java中数据的运算是通过补码进行计算的。

面向对象

  • 本文主要从以下几个方面去讲述面向对象
    • 从面向过程到面向对象
    • 面向对象的特征
    • 面向对象的原则
  • 在面向对象没有出现之前，人们主要使用面向过程编程的范式，以函数，方法，变量为媒介，定义事件一步一步的执行流程。着重点不在具体的事物而在具体的流程上。就拿把大象放在冰箱中说明，众所周知，把大象放在冰箱中需要三步，打开冰箱门，把大象放在冰箱中，关上冰箱门。这是一个典型的面向过程的案例。而面向对象的关注点则不同，面向对象的主体是事物，在上述例子中，大象和冰箱是两个需要处理的对象。大象的体积（属性）和冰箱的行为（打开，闭合）是主要的关注点。

  • 面向对象主要有以下几个特征:封装，继承，多态，抽象。下面我将逐一说明。

    • 首先是封装，字面意思理解，就是将类封闭起来，那么封闭起来的意义在于限制类的使用。在Java中，可以用属性和行为描述一个类，即封装的意义在于将类的属性和行为提供给不同的人使用，也就是权限控制。主要通过访问修饰符和包来体现；
    • Java中主要有四种访问修饰符，分别是public,private,default,proteced

    • public的意思是所有人都可以访问，private表示只有自己(本类)可以访问，default表示包访问权限(也就是本包内的类都可以访问)，protected除了private权限外还提供了继承访问。具体代码如下:

    • 然后我们来说说抽象，抽象相对比较好理解，就是将信息整合，隐藏信息，保护数据，主要是方法层面。在Java中主要以抽象类和接口的形式存在。

    • 在Java中抽象类不能被实例化，只能被继承。抽象类可以包含属性与方法，方法可以有具体实现，不包含具体实现的方法叫做抽象方法；继承抽象类必须实现所有抽象方法，更有利于代码复用。

  • 面向对象的主要原则为SOLID,分别是单一职责原则，开笔原则，里氏替换原则，接口隔离原则和依赖反转原则，下面将逐一介绍：

    • 单一职责原则（SRP）:一个类或者模块之负责完成一个职责（或者功能）
    • 开闭原则(OCP):软件实体(模块，类，方法等)应该“对扩展开放，对修改关闭”
    • 里氏替换原则(LSP):子类对象能够替换程序中父类对象出现的任何地方，并且保证原来程序的逻辑行为不变及正确性不被破坏
    • 接口隔离原则(ISP):客户端不应该被强迫依赖它不需要的接口
    • 依赖反转原则(DIP):高层模块不要依赖底层模块。高层模块和底层模块应该通过抽象来相互依赖。除此之外，抽象不要依赖具体实现细节，具体实现细节依赖抽象。

补充说明

  • 面向对象与抽象:
    • 我们认识自然是通过将其分门别类，通过界门纲目科属种的形式来达成共识，例如将人，猿，猩猩等抽象成灵长目。计算机语言也是一样，最初，人们只能通过0，1一样的二进制，进一步对底层进行抽象形成汇编语言，C语言，Java，python等。有些是对机器的抽象，有些是对待解决问题的抽象，在Java(OOP)中将这种解空间的元素就叫做对象。即Java是通过将事物进行属性和行为的刻画，将其抽象成一个个的模型，然后通过对模型属性以及行为的调度，去解决相应的问题。OOP中对象最重要的三点为:属性，行为，(地址)标识。

优化篇

  • 范式

    • 键的分类
      • 超键:能唯一标识元组的属性集
      • 候选键: 如果超键不包含多余的属性，那么这个超键就是候选键
      • 主键: 用户可以从候选键中选择一个作为主键
      • 外键: 如果数据表中某个属性集不是其主键，而是另一个数据表的主键，那么这个属性集就是这个数据表的外键
      • 主属性:包含在任意候选键中的属性
      • 非主属性: 不包含在任意候选键中的属性
    • 范式分类
      • 1NF: 数据库表中任意属性都是原子性的，不可再分。
      • 2NF: 数据表李的非主属性要和这个数据表的候选键有完全依赖关系
      • 3NF: 对于任何非主属性都不传递依赖于候选键
      • BCNF: 消除主属性对候选键的部分依赖或者传递依赖关系

  • 索引

    • 帮助数据库管理系统高效获取数据的数据结构
    • 分类
      • 普通索引:没有约束，提高效率
      • 唯一索引:数据唯一性约束，多个唯一性索引
      • 主键索引:在唯一性索引基础上不为空
      • 全文索引

基础篇

  • SQL的分类详解

    • SQL按照功能划分可以分为以下几种:
      • 负责数据库和表的创建工作的DDL（数据库定义语言）
      • 负责数据的增删改的DML（数据库管理语言）
      • 负责数据库权限访问控制的DCL(数据库控制语言)
      • 负责数据库查询的DQL(数据库查询语言)

  • DBMS的分类详解

    • DBMS，数据库管理系统是我们管理数据库的工具。根据不同的使用场景可以分为以下几种类型:
      • 关系型（实体-对象）
      • 键值型
      • 文档型
      • 搜索引擎
      • 列式数据库
      • 图形数据库

  • SQL是如何执行的？

    • 我们首先跳出这个问题，先来看这样一个场景，我和你说：“帮我带瓶水”。我们将这句话比做一条SQL,那么我编写完一条SQL语句，之后，首先要做的就是将它“说”出来，即从客户端传递给服务器端。这样就需要一个接收方，在mysql中承担这个任务的就是连接层。
    • 当你听到我说“帮我带瓶水”之后，你就会对这句话进行分析，首先是语义语法分析，也就是这句话是什么意思，然后就是听到这句话的你需要做什么，这就是优化器（在mysql中就是选择合适的索引和连接），之后就是对象分析（权限扽袭），“我”作为说话的人和你的关系，是朋友还是陌生人。最后就是执行。
    • 这个过程在mysql的执行流程中具体是这样的形式，首先SQL从客户端发送给服务器，由连接层接受，连接层将SQL移交给SQL层，SQL层中解析器进行语法语义分析，这句话有没有错误，然后提交给优化器确定怎么去执行，最后提交给执行器，判断是否有权限执行，如果有就直接执行。

  • 存储引擎

    • mysql的存储引擎采用插件类型，根据不同的场景可以采用不同的存储引擎，以下便是不同存储引擎和他们之间的区别：

      • MylSAM:5.5之前默认存储引擎，不支持事务，速度快，占用资源少
      • Memory:系统内存作为存储介质。

          • 两张表的外连接，会有一张是主表，另一张是从表。

        • 外连接：会返回一个表中的所有记录，以及另一个表中匹配的行，包括左外连接，右外连接和全连接

          • 事务提交之后对数据的修改是持久性的

        • 幻读：有多个事务的情况下，事务查询结果前后不同（主要是insert）
      • 隔离级别
        • 读未提交
        • 读已提交：避免脏读
        • 可重复读：避免不可重复读，保证事务在相同查询条件下两次查询结果一致（mysql）
