    • 首先2的8次方为256，即byte的取值范围为正负256之间，由于0的存在，所以在-128—-127或者-127—-128；我们知道Java中所有正数都是有符号位的，8位符号位最大值为0111 1111 为127，1111 1111为-127；现在假设byte占用9位而不是8位，那么如何表示+(-)128呢？ 0000 1000 0000 为128； 0001 1000 0000为-128；正数的补码为其本身，负数的补码运算先将原码符号位不变，其余取反，然后在其(反码)基础之上+1；即0001 0111 1111 + 1 也就是0001 1000 0000 与其原码相同。所以计算机就取1000 0000 -0为-128；

数据库概念盘点（二）

发表于 2020-10-19
本文字数： 700 | 阅读时长 ≈ 1 分钟

优化篇

  • 范式

    • 键的分类
      • 超键:能唯一标识元组的属性集
      • 候选键: 如果超键不包含多余的属性，那么这个超键就是候选键
      • 主键: 用户可以从候选键中选择一个作为主键
      • 外键: 如果数据表中某个属性集不是其主键，而是另一个数据表的主键，那么这个属性集就是这个数据表的外键
      • 主属性:包含在任意候选键中的属性
      • 非主属性: 不包含在任意候选键中的属性
    • 范式分类
      • 1NF: 数据库表中任意属性都是原子性的，不可再分。
      • 2NF: 数据表李的非主属性要和这个数据表的候选键有完全依赖关系
      • 3NF: 对于任何非主属性都不传递依赖于候选键
      • BCNF: 消除主属性对候选键的部分依赖或者传递依赖关系

  • 索引

    • 帮助数据库管理系统高效获取数据的数据结构
    • 分类
      • 普通索引:没有约束，提高效率
      • 唯一索引:数据唯一性约束，多个唯一性索引
      • 主键索引:在唯一性索引基础上不为空
      • 全文索引

基础篇

  • SQL的分类详解

    • SQL按照功能划分可以分为以下几种:
      • 负责数据库和表的创建工作的DDL（数据库定义语言）
      • 负责数据的增删改的DML（数据库管理语言）
      • 负责数据库权限访问控制的DCL(数据库控制语言)
      • 负责数据库查询的DQL(数据库查询语言)

  • DBMS的分类详解

    • DBMS，数据库管理系统是我们管理数据库的工具。根据不同的使用场景可以分为以下几种类型:
      • 关系型（实体-对象）
      • 键值型
      • 文档型
      • 搜索引擎
      • 列式数据库
      • 图形数据库

  • SQL是如何执行的？

    • 我们首先跳出这个问题，先来看这样一个场景，我和你说：“帮我带瓶水”。我们将这句话比做一条SQL,那么我编写完一条SQL语句，之后，首先要做的就是将它“说”出来，即从客户端传递给服务器端。这样就需要一个接收方，在mysql中承担这个任务的就是连接层。
    • 当你听到我说“帮我带瓶水”之后，你就会对这句话进行分析，首先是语义语法分析，也就是这句话是什么意思，然后就是听到这句话的你需要做什么，这就是优化器（在mysql中就是选择合适的索引和连接），之后就是对象分析（权限扽袭），“我”作为说话的人和你的关系，是朋友还是陌生人。最后就是执行。
    • 这个过程在mysql的执行流程中具体是这样的形式，首先SQL从客户端发送给服务器，由连接层接受，连接层将SQL移交给SQL层，SQL层中解析器进行语法语义分析，这句话有没有错误，然后提交给优化器确定怎么去执行，最后提交给执行器，判断是否有权限执行，如果有就直接执行。

  • 存储引擎

    • mysql的存储引擎采用插件类型，根据不同的场景可以采用不同的存储引擎，以下便是不同存储引擎和他们之间的区别：
      • innodb:5.5之后的默认存储引擎，最大特点是支持事务，行级锁定，外键约束。
      • MylSAM:5.5之前默认存储引擎，不支持事务，速度快，占用资源少
      • Memory:系统内存作为存储介质。

        • 外连接（左连接，右连接）

          • 两张表的外连接，会有一张是主表，另一张是从表。
          • 多张表的外连接，第一张是主表（显示全部行），剩下的表显示对应连接的信息

          • 左外连接

            • 左边的表是主表，需要显示左边表的全部行，而右侧的表是从表(+)表示哪个是从表

      • 连接分类

        • 内连接：将多个表之间满足连接条件的数据行查询出来，包括等值连接，非等值连接和自连接
        • 外连接：会返回一个表中的所有记录，以及另一个表中匹配的行，包括左外连接，右外连接和全连接
        • 交叉连接（笛卡尔积）：返回左表中的每一行与右表中的每一行的组合。

      • 存储类型

        • IN:向存储过程传入参数，存储过程中修改该参数的值，不能被返回
        • OUT: 把存储过程计算的结果放到该参数中，调用这可以得到返回值
        • INOUT: 上述两者结合

    • 事务

      • 特性ACID
        • 原子性Atomicity
        • 一致性Consistency
          • 数据库进行事务操作后，会由原来的一致状态编程另一种一致的状态
        • 隔离型Isolation
          • 每个事务都有独立的，不受其他事务的执行影响，也就是事务在提交之前对其他事务不可见
        • 持久性Durabitlty
          • 事务提交之后对数据的修改是持久性的
      • 命令

    • 隔离级别

      • 常见异常类型
        • 脏读：读到了其他事务还没有提交的数据
        • 不可重复读:对某数据进行读取，发现两次读取的结果不同，也就是说没有读到相同的内容。这是因为其他事务对这个数据进行了修改或者删除(主要是update和delete)
        • 幻读：有多个事务的情况下，事务查询结果前后不同（主要是insert）
      • 隔离级别
        • 读未提交
        • 读已提交：避免脏读
        • 可重复读：避免不可重复读，保证事务在相同查询条件下两次查询结果一致（mysql）
