高性能MySQL读书笔记(八)

发表于 2020-03-29 | 分类于 mysql
本文字数： 1.3k | 阅读时长 ≈ 1 分钟

复制的常见问题

  • 数据损坏或丢失错误
    • 主库意外关闭
    • 备库意外关闭
    • 主库二进制日志损坏
    • 备库中继日志损坏
    • 二进制日志与InnoDB事务日志不同步
  • 使用非事物型表
  • 混合事物型和非事务型表
  • 不确定语句
  • 主库和备库使用不同的存储引擎
  • 备库发生数据改变
  • 不唯一的服务器ID
  • 未定义的服务器ID
  • 对未复制数据的依赖性
  • 丢失的临时表
  • 不复制所有的更新
  • InnoDB加锁读引起的锁争用
  • 在主-主复制结构中写入两台主库
  • 过大的复制延迟
    • 产生延迟方式
      • 突然产生延迟再跟上
        • 由于一条运行时间很长的查询导致
      • 稳定的延迟增加
        • 没有长时间运行查询时也会出现
    • 备库优化方式(禁止额外工作，减少备库的负载)
      • 不要重复写操作中代价较高的部分
      • 在复制之外并行写入
      • 为复制线程预取缓存
  • 来自主库过大的包
  • 受限制的复制带宽
  • 磁盘空间不足

MySQL复制的一些高级特性

  • 半同步复制
    • 确保备库拥有主库数据的拷贝，减少潜在的数据丢失危险
  • 复制心跳
    • 保证备库一直与主库相关联，避免悄无声息的断开连接

总结

  • 使用复制时需要注意项
    • 使用Percona Toolkit的pt-table-checksum以确定备库是主库的真实拷贝
    • 监控复制以确定其正在运行并且没有落后于主库
    • 理解复制的异步本质，并且设计你的应用以避免或容忍从备库读取脏的数据
    • 在一个复制topo中不要写入超过一个服务器，把备库配置为只读，并降低权限以组织对数据的改变
