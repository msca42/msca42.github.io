Java基础(2)

发表于 2021-06-04 | 更新于 2021-06-05 | 分类于 Java
本文字数： 1.8k | 阅读时长 ≈ 2 分钟
  • 编程语言首先是运行在计算机系统上的一种人与计算机交流的工具。那么如何将人类现实种的问题编程计算机能够理解的语言就是我们需要做的。这一过程叫做抽象；
  • 例子: 汇编语言是对底层机器的抽象，C是对汇编的抽象。
  • OOP所做的就是从现实问题出发，将现实问题变成一个个的对象模型。通过对对象的操作，来进行对话。
  • 如何在“问题空间”（问题实际存在的地方）的元素与“方案空间”（对实际问题进行建模的地方，如计算机）的元素之间建立理想的“一对一”的映射关系？
  • 面向对象编程设计方法
    • 万物皆对象
    • 程序是一组对象，通过消息传递来告知彼此该做什么
    • 每个对象都有自己的存储空间，可容纳其他对象
    • 每个对象都有一种类型
    • 同一类所有对象都能接受相同消息
  • 我们把所有问题模型抽象成对象模型。
    • 问题模型中有不同的事物，事物有不同的属性和行为。
    • 问题模型中的事物就对应不同的对象；对象就存在自己的属性和行为，并且不同的事物代表不同的类型。
    • 不同事物之间的交流行为对应对象之间的交流，这就需要一个媒介告知对象，这个媒介就是消息。
    • 不同对象之间的关系，包含，重叠，交叉。对应的就是组合，聚合，关联，依赖。所谓组合，就是整体的不可缺少部分，聚合就是整体中可以缺少部分。
    • 如果是包含关系，消息的传递是点对点，还是点对多点。传递消息和接受消息的基本单位是对象，那么相同类型的对象接受到信息的点就是一样的。
  • 对象(Grady Booch): 一个对象具有自己的状态，行为和标识。这意味这对象有自己的内部数据（提供状态），方法（产生行为），并彼此区分（每个对象在内存中都有唯一的地址）。
  • 对象是对现实问题的抽象，现实问题中存在不同的事物，生物学中才用界门纲目科属种的不同类型来区分生物，同样，在计算机中模拟的对象也具有不同的类。类是对象的高度概括，通过类，我们才能知道对象，产生对象。
  • 对象之间通过消息来进行交流，那么消息是如何定义的？以及请求是如何发生的。在Java中我们借助了接口来定义对象发出的请求。接口的类行为的标准模板，类是接口的实现方式。类中的行为受到接口的影响，并通过类的对象进行调用。
  • 访问控制：public ,private,protected,default
  • 当我们拥有一个类的时候，如果此时出现了一个类似的需求，不过更加的细微，我们这时候就可以采取继承的方法。通过继承创建派生类。也就是说当基类无法满足需求的时候，此时基类就是一个框架，我们添加新的行为来满足我们更加精细化的需求，也就是纯粹替换和扩展替换，用范式表达就是里氏替换原则。
  • 什么是多态，程序不知道接受的是什么类型，但能够执行正确的形式就称为多态。在非OOP中，才用早期绑定，编译器生成绝对函数名的调用，然后将该调用地址解析为代码的绝对地址。而在OOP中，才用后期绑定，被调用代码直到运行时才确定，编译器确保方法存在，并对参数和返回值执行类型检查。
  • 参数化类型机制，也即是泛型产生的原因，是因为在此之前，在集合中存储的都是object类型，这里有两个概念要提及。当我们对某个派生类使用多态时，我们清楚的直到它的父类是object,这叫做向上转型，而当我们想要将object类型中转化为具体类型时，这就出现了安全问题，这叫做向下转型。我们不清楚我们需要转化的具体类型是什么？这样做会导致运行时异常。
  • 在谈论对象的生命周期之前，有必要概述下计算机的硬件基础。首先，我们直到计算机主要是通过输入设备，输出设备，系统单元和外部存储装置。具体而言，外在的有键盘鼠标，显示器，电源等，内在的有主板，内存，硬盘，cpu,显卡，网卡等。cpu是计算机的主脑，负责连接统筹其他所有装置，它与其他设备连接主要通过南桥北桥，由于技术进步，所有北桥被集成在cpu中。北桥主要连接内存等快速设备，南桥主要连接硬盘等慢速设备。cpu中主要包含两个部分，算数逻辑单元和控制单元以及多级缓存。
